1 - Installation

Pour installer FastChangeLog, décompressez l'archive.

Editer le fichier index.php et à la ligne 34 :

$monTheme->addVar("URL", "http://localhost/changelog") ;

rempacer http://localhost/changelog par votre url d'accès.

Ensuite, il suffit de personnaliser les fichiers de theme, contenu dans le répertoire themes.
Enfin, copier les fichiers (hormis répertoire doc et le contenu du répertoire changelog) sur votre serveur.

2 - Personnalisation

Pour personnaliser l'affichage de votre changelog, lisez la documentation fasttemplate.htm.

Les fichiers de thèmes sont :
- bugfix.htm : contenant la ligne d'affichage pour une résolution d'un bug via un module spécial.
- index.htm : page d'index
- ligne.htm : ligne normal
- retour.htm : lien de retour si on affiche le changelog d'une version
- titre-major.htm : affichage de la version majeur (e.g. version 2)
- titre-minor.htm : affichage de la version mineur (e.g. version 2.1.2)

2.1 - index.htm

{ URL } : contient l'url pour accèder au changelog
{ PAGE } : va contenir soit la liste des versions majeurs, soit la liste des versions mineurs avec les change

2.2 - titre-majeur.htm

{ URL } : contient l'url pour accèder au changelog
{ VERSION } : contient le numéro de version majeur

2.3 - titre-mineur.htm

{ VERSION } : contient le numéro de version mineur
{ NAME } : nom de la personne qui a validé la version
{ DATE } : date de la version
{ CHANGE } : contient les lignes de changements

2.4 - ligne.htm

{ TEXTE } : texte du changement

2.5 - bugfix.htm

{ BUG } : contient le numéro du bug
{ COMPLEMENT } : complement d'information

2.6 - retour.htm

{ URL } : contient l'url pour accèder au changelog

3 - Création des versions

Pour créer une version majeur, il suffit de créer un répertoire dans le répertoire changelog qui à pour nom le numéro de la version.
Pour créer une version mineur, il suffit de créer un répertoire dans le répertoire de version majeur qui a pour nom tout la version (majeur + mineur).
Dans ce dernier répertoire, il suffira de créer et remplir un fichier appelé log.txt

4 - Format du fichier log.txt

Le fichier log.txt à une structure un peu spécial.
La première ligne contient le nom de la personne qui à créer le changelog.
La deuxième ligne contient la date du changelog.
Ensuite, tout le reste du fichier contient le changelog.

4.1 - Gestion des bugfix

Une fonction spéciale pour le bugfix a été créé. Il suffit que la ligne commence par @, suivi du numéro du bug.
Si vous souhaitez ajouter des informations complémentaires, ajoutez les après avoir laissé un espace.
